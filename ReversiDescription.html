General descrption

Webec graded exercise: Reversi

Provides a service to play reversi against a bot or another player

Developed by Claudio Seitz, May 2017

Backend

Web server

Unique web server, all clients connect to this server

Clients and web server communicates via web sockets and json messages

Handles and validates all messages from each client

Board

Validates the incoming action

handles the game logic

Bot

Evaluates his best action using a sounding algorithm

Sounding algorithm

Implemented concurrently

Looks at which action the bot has the most likely opportunity to win the game

Calculates 4000 variants per action

Pvp

Matches a white waiting player with a black waiting player and starts a new pvp round

Round

Handles the game sequence

Sends json messages to the clients

Bot round

Player vs bot

Pvp round

Player vs player

Handles the spacial cases, if there is a problem of one player

Frontend

Connection

Handles incoming server json messages

Sends client actions to the server as json message

Game

Visualizes the incoming json messages

UserHandler

Handles the user action

TokenHandler

Handles the behaviour of a token
